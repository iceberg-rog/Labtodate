## Summary

On arrival, `tsc --noEmit` failed across ~34 files. Root cause: a filesystem
corruption event (working-tree mtime ~today) that hit **35 source files in two
distinct ways**:

1. **17 files — trailing NUL-byte padding.** Content fully intact; a run of
   `0x00` bytes was appended past EOF (TS1127 "Invalid character"). 16 of the 17
   stripped to **byte-for-byte identical to git HEAD**; the 17th
   (`marketplace/[slug]/page.tsx`) carried the BUG-024 fix and was complete.
2. **17 files — hard truncation (content loss).** Files cut mid-statement
   (unterminated literals / missing close tags). 14 were an exact **prefix of
   HEAD** (no local edits → lossless HEAD restore); 3 carried working-copy
   security fixes and were recovered from the **git stash**
   `user-local-edits-pre-server-sync` (clean git blobs, immune to FS corruption):
   `admin/actions.ts`, `quotes/actions.ts`, `api/auth/[...all]/route.ts`.

Two further defects surfaced during recovery:

- **Pre-existing broken commit in HEAD:** `admin/actions.ts` (identical in HEAD
  and stash) contained a **duplicated `rejectPayment` body** (lines 2658–2704) —
  a botched double-append from the documented 2026-06-08 BUG-026 Edit-tool
  truncation recovery. It made the file non-compiling. HEAD never built. Removed
  the duplicate; function now closes correctly at line 2657.
- **Wiped untracked component:** `companies/page.tsx` imports
  `@/components/admin/BrowseSupplierButton`, whose source file never existed in
  git and was destroyed by the corruption. Reconstructed faithfully as a thin
  client wrapper around the existing `ShopBrowser` component (admin-only; no new
  server actions, no data mutations, no manual-payment/Stripe surface).

Also re-applied a **regressed** prior fix:

- **BUG-006 (invariant A12):** ledger lists password min length as 12, but HEAD
  shipped `minPasswordLength: 8` (regressed in an earlier server sync). Restored
  to `12`. Affects only newly-set passwords; no existing user is locked out.

All recovered/edited content was verified intact: every security-fix marker is
present post-recovery (BUG-009/018/022/023/024/025/026), and `tsc --noEmit` = 0.
